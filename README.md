# ValuesBot

Telegram-бот для прохождения теста на ранжирование личных ценностей. На каждом шаге пользователь выбирает, какая из двух ценностей для него важнее. Итог — упорядоченный список всех 30 ценностей от самой важной к наименее важной.

## Как это работает

Бот использует **пошаговую сортировку слиянием (merge sort)**: каждое нажатие кнопки выполняет одно попарное сравнение. Для 30 ценностей потребуется примерно **119 сравнений** в худшем случае.

Прогресс сохраняется в SQLite — можно закрыть чат и продолжить позже. Доступны отмена последнего выбора и перезапуск теста.

## Требования

- Python 3.12
- [uv](https://docs.astral.sh/uv/) (рекомендуется)

## Установка

```bash
git clone <repository-url>
cd ValuesBot
uv sync
```

## Настройка

1. Создайте бота у [@BotFather](https://t.me/BotFather) в Telegram и получите токен.
2. Скопируйте файл окружения:

```bash
cp .env.example .env
```

3. Укажите токен в `.env`:

```env
BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
```

## Запуск

```bash
uv run python main.py
```

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Начать тест или продолжить незавершённый |
| `/restart` | Начать тест заново (с подтверждением) |
| `/result` | Показать итоговый рейтинг ценностей |

## Структура проекта

```
ValuesBot/
├── bot/
│   ├── config.py      # конфигурация и пути
│   ├── values.py      # загрузка ценностей из data/values.txt
│   ├── sort.py        # инкрементальный merge sort
│   ├── db.py          # хранение сессий в SQLite
│   ├── keyboards.py   # инлайн-клавиатуры
│   ├── texts.py       # тексты сообщений
│   ├── handlers.py    # обработчики команд и callback
│   └── bot.py         # точка входа бота
├── data/
│   └── values.txt     # список из 30 ценностей
├── main.py
├── pyproject.toml
└── .env.example
```

## Формат ценностей

Каждая строка в `data/values.txt` имеет вид:

```
НАЗВАНИЕ (пояснение)
```

В сообщении бота отображается полное пояснение, на кнопках — только короткое название в sentence case (например, «Социальная сила»).
